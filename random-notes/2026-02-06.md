	- **Room** `**@Upsert**`
	- Updates **only when the PRIMARY KEY matches**
	    
	- **Does NOT** update based on a `unique index`
	    
	- To update by `phoneNumber`, you must use a **separate UPDATE query**
	    
	
	- **Correct way to update a contact**
	    
	
	- Use `UPDATE contacts SET name = ... WHERE phoneNumber = ...`
	    
	- Return `Int` to know whether a row was updated
	    
	
	- **Unique index vs Primary Key**
	    
	
	- `unique index` ‚Üí prevents duplicates
	    
	- `primary key` ‚Üí what Room uses for UPDATE vs INSERT
	    
	- `@Upsert` checks **only the primary key**

	- UI delete is unreliable ‚Üí use **SQL console**
