2. **GraphQL in Android**
	- You getÂ **exactly what you ask for.**
	- GraphQL uses a **single endpoint** to fetch exactly what you need in a single request.
	- It lets clients (your Android app) request exactly the data they need, nothing more, nothing less.
	- Key Advantages:
		- Fetch only what you need: Avoid over-fetching or under-fetching data.
		- Single endpoint: Simplifies server communication.
		- Strongly typed schema: Backend defines exactly what queries are allowed.
		- Real-time support: With subscriptions, you can get live updates.
	- Basic Concepts of GraphQL
		- **Query**: Read or fetch data.
		- **Mutation**: Create, update, or delete data.
		- **Subscription**: Listen to real-time data changes (like notifications).
	- Use **Apollo** Kotlin
		- Apollo generates Kotlin models from GraphQL queries, giving you typesafe code and easy integration
