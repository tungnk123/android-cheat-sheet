1. Parallel API Calls in Kotlin: async vs Flow.zip vs Flow.combine (Complete Guide)
	- async:
		- async is used to run tasks **in parallel** and give you a **Deferred** result that you can later await().
		- One-time API calls  
		- Heavy background work you want to run together
	- Flow.zip
		- zip waits for **one value from each Flow**, and then combines them **pair by pair**.
		- When all sources must emit **matching items**  
		- One-shot flows (like API calls converted into flows)
	- Flow.combine
		- combine emits a new value **whenever any Flow updates**, using the **latest values** from all flows.
		-  Live updates 
		- UI that must react anytime any data source changes 
		- Multiple continuous streams
	- **async →** “start now, collect later.”
	- **zip →** “wait for everyone, then emit once.”
	- **combine →** “emit whenever anyone updates.”
2. **How to use Mapbox’s default user-location icon**
	- Mapbox’s **default blue location dot cannot be used with PointAnnotation**.
	- To show the **default icon**, you must use **Location Component**.
	- Remove your custom user marker logic (`updateUserMarker` + PointAnnotation).
	- Enable the location component inside `loadStyle`:
	
	```
	binding.mapView.mapboxMap.loadStyle(Style.TRAFFIC_NIGHT) {
	    val locationPlugin = binding.mapView.location
	    locationPlugin.updateSettings {
	        enabled = true
	        pulsingEnabled = true
	    }
	}
	```
	
	→ Mapbox then displays and auto-updates the **built-in blue dot** (best performance & correct behavior).
