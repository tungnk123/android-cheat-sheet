
 1. **⚔️ Hilt vs Koin — The Core Difference**
	- **Hilt (Dagger-based)** → _Compile-time DI_  
	    Generates dependency graphs before runtime using annotation processing.  
	    ✅ Fast startup, compile-time safety, zero reflection.  
	    ⚠️ More boilerplate and longer build times.
	    
	- **Koin** → _Runtime DI_  
	    Builds dependency graphs dynamically via Kotlin DSL.  
		    ✅ Simple syntax, no annotation processing.  
	    ⚠️ Slower startup, runtime resolution errors, reflection overhead.
	
	=> Hilt writes your dependency graph _at build time._  
	Koin builds your dependency graph _when your app runs.

2. **Multiplatform resources**
	- Compose Multiplatform provides a special `compose-multiplatform-resources` library and Gradle plugin support for accessing resources in common code across all supported platforms.
	- Create a new directory `composeResources` in the source set directory you want to add the resources to
	- Organize the `composeResources` directory structure according to these rules:
		- Images should be in the `drawable` directory.
		- Fonts should be in the `font` directory.
		- Strings should be in the `values` directory.
		- Other files should be in the `files` directory, with any folder hierarchy you may find appropriate.
	- Qualifiers:
		- The same resource should be presented in **different ways** depending on the environment, such as **locale**, **screen density**, or **interface theme**.
		- Add qualifiers to directory names using a **hyphen**
		- The library supports (in the order of priority) the following qualifiers: [language](https://www.jetbrains.com/help/kotlin-multiplatform-dev/compose-multiplatform-resources-setup.html#language-and-regional-qualifiers), [theme](https://www.jetbrains.com/help/kotlin-multiplatform-dev/compose-multiplatform-resources-setup.html#theme-qualifier), and [density](https://www.jetbrains.com/help/kotlin-multiplatform-dev/compose-multiplatform-resources-setup.html#density-qualifier).

	- Usage:
		- Build the project to generate the special `Res` class which provides access to resources.
		- Images:
			- fun painterResource(resource: DrawableResource): Painter {...}
			- fun imageResource(resource: DrawableResource): ImageBitmap {...}
			- fun vectorResource(resource: DrawableResource): ImageVector {...}
		- Icons:
			- painter = painterResource(Res.drawable.ic_sample_icon),
			- olorFilter = ColorFilter.tint(Color.Blue)
		- Strings:
			- Store all string resources in XML files in `composeResources/values` directories.
			- fun stringResource(resource: StringResource): String {...}
			- suspend fun getString(resource: StringResource): String
		- Strings arrays:
			- fun stringArrayResource(resource: StringArrayResource)
			- suspend fun getStringArray(resource: StringArrayResource):
		- Fonts:
			- Store custom fonts in the `composeResources/font` directory as `*.ttf` or `*.otf` files.
			- To load a font as a `Font` type, use the `Font()` composable function
		- Raw files:
			- To load any raw file as a byte array, use the `Res.readBytes(path)` function
